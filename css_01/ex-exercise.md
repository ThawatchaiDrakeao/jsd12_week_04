### 1. Inline Style (ทาสีเป็นจุดๆ)
คือการเขียนคำสั่งตกแต่งลงไปในตัว Tag HTML นั้นๆ โดยตรงโดยใช้ attribute `style` เหมาะสำหรับแก้จุดเล็กๆ น้อยๆ แบบด่วนๆ

* **เปรียบเทียบ:** เหมือนคุณถือแปรงทาสี เดินไปที่เสาต้นหนึ่งแล้วระบายสีลงไปแค่เสาต้นนั้นต้นเดียว
* **ตัวอย่างโค้ด:**
    ```html
    <h1 style="color: blue; font-size: 20px;">นี่คือการเขียนแบบ Inline</h1>
    ```

### 2. Internal Style (ทาสีรวมในใบสั่งเดียว)
คือการรวมคำสั่งตกแต่งทั้งหมดไว้ที่ส่วนหัวของไฟล์ HTML (ภายใน Tag `<style>`) โดยปกติจะวางไว้ในส่วน `<head>`

* **เปรียบเทียบ:** เหมือนคุณเขียนโน้ตแปะไว้ที่ประตูบ้านว่า "ในบ้านหลังนี้ ให้เสาทุกต้นเป็นสีฟ้า และกำแพงเป็นสีขาว" (มีผลเฉพาะกับหน้า HTML หน้านั้นหน้าเดียว)
* **ตัวอย่างโค้ด:**
    ```html
    <head>
      <style>
        
        h2 
        {

          color: green;
          text-decoration: underline;

        }
        
      </style>
    </head>
    <body>
      <h2>นี่คือการเขียนแบบ Internal</h2>
    </body>
    ```

### 3. External Style (คู่มือทาสีจากบริษัทแม่)
คือการแยกโค้ด CSS ออกไปเป็นไฟล์ใหม่ (นามสกุล `.css`) แล้วค่อยใช้ Tag `<link>` เชื่อมโยงไฟล์นั้นเข้ามาใน HTML วิธีนี้เป็นวิธีมาตรฐานที่มือโปรใช้กัน เพราะจัดการง่ายและใช้ไฟล์เดิมตกแต่งได้หลายหน้า

* **เปรียบเทียบ:** เหมือนคุณมี "คู่มือมาตรฐานการตกแต่ง" แยกต่างหาก ถ้าคุณสร้างบ้าน 10 หลัง คุณก็แค่เอาคู่มือเล่มเดิมไปใช้กับทุกหลัง บ้านทุกหลังก็จะดูเหมือนกันทันที
* **ตัวอย่างโค้ด:**
    * **ไฟล์ `style.css`:**
        ```css
        p {
          color: red;
          font-family: Arial;
        }
        ```
    * **ไฟล์ `index.html`:**
        ```html
        <head>
          <link rel="stylesheet" href="style.css">
        </head>
        <body>
          <p>นี่คือการเขียนแบบ External</p>
        </body>
        ```

### สรุปความแตกต่าง

| วิธีการ          | ความสะดวก     | การนำไปใช้ซ้ำ | ความเป็นระเบียบ |
| :---          | :---           | :---        | :---         |
| **Inline**    | เร็วที่สุดในจุดเดียว | ต่ำมาก (ต้องไล่แก้ทีละจุด) | รก (ปนกับเนื้อหา) |
| **Internal**  | ดีสำหรับหน้าเดียว  | ปานกลาง (เฉพาะในหน้านั้น) | ดีขึ้น |
| **External**  | ช้าตอนเริ่ม แต่ดีที่สุดยาวๆ | สูงมาก (ใช้ได้ทั้งเว็บไซต์) | ดีที่สุด (แยกเนื้อหาออกจากดีไซน์) |
